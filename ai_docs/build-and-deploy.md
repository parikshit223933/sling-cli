# Build and Deploy Guide

## Overview

This repo is a fork of `slingdata-io/sling-cli` with custom transform implementations. The binary is built from source and deployed to the EC2 instance at `/usr/local/bin/sling`.

---

## Making Changes

### 1. Edit code locally

All custom code lives in `core/dbio/iop/transforms_impl.go`. If adding new transform types, register them in this file's `init()` function.

The original stubs are in `core/dbio/iop/transforms.go` — do NOT edit this file. Our `transforms_impl.go`'s `init()` overrides the stub function variables at startup.

### 2. Test build locally

```bash
cd ~/Desktop/opensource/sling-cli
go build -mod=vendor -o sling-custom ./cmd/sling/
./sling-custom --version
```

If you added new dependencies:
```bash
go mod tidy
go mod vendor
go build -mod=vendor -o sling-custom ./cmd/sling/
```

### 3. Commit and push

```bash
git add -A
git commit -m "Description of change"
git push origin main
```

### 4. Pull and build on server

```bash
# SSH to server
ssh -A -J ubuntu@13.202.163.240:443 ubuntu@ip-172-31-99-217.ap-south-1.compute.internal -i ~/Desktop/pem_key_files/ninjas-production.pem

# On the server:
cd ~/sling-cli
git pull
export PATH=$PATH:/usr/local/go/bin
go build -mod=vendor -o sling-custom ./cmd/sling/
```

### 5. Replace the system binary

```bash
# Stop any running sync first
pkill -f 'sling run' 2>/dev/null

# Replace binary
sudo cp ~/sling-cli/sling-custom /usr/local/bin/sling

# Verify
sling --version
# Should output: Version: dev
```

### 6. Restart sync

```bash
cd ~/sling
./scripts/sync.sh lsq
```

## Complete Example: Adding a New Transform

Say you want to add a `mask_email` transform that replaces `user@domain.com` with `u***@domain.com`:

### Step 1: Register the transform in `transforms.go` (upstream file)

Check if it already exists in `TransformsLegacyMap`. If not, add it:

```go
// In core/dbio/iop/transforms.go, in the init() function:
TransformsLegacyMap["mask_email"] = TransformLegacy{
    Name: "mask_email",
    FuncString: func(sp *StreamProcessor, val string) (string, error) {
        parts := strings.SplitN(val, "@", 2)
        if len(parts) != 2 {
            return val, nil
        }
        masked := string(parts[0][0]) + "***@" + parts[1]
        return masked, nil
    },
}
```

Since we don't modify upstream files, add this registration in `transforms_impl.go`'s `init()` instead.

### Step 2: Build and test locally

```bash
go build -mod=vendor -o sling-custom ./cmd/sling/
```

### Step 3: Use in config

```yaml
streams:
  my_table:
    transforms:
      - email_column: mask_email
```

### Step 4: Deploy (steps 3-6 from above)

## Troubleshooting

### Build fails with "no matching files found"

```bash
mkdir -p vendor/github.com/substrait-io/substrait/tests/cases/dummy
touch vendor/github.com/substrait-io/substrait/tests/cases/dummy/dummy.test
```

### Build fails with "replacement directory does not exist"

The `go.mod` has local replace directives commented out. If they get uncommented, comment them again:

```go
// replace github.com/flarco/g => ../g
// replace github.com/slingdata-io/sling => ../sling-lib
```

### Binary shows wrong version

The custom build always shows `Version: dev`. This is expected — the official release injects version via build flags that we don't replicate.

### Server doesn't have Go installed

```bash
wget -q https://go.dev/dl/go1.24.4.linux-arm64.tar.gz
sudo tar -C /usr/local -xzf go1.24.4.linux-arm64.tar.gz
rm go1.24.4.linux-arm64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
export PATH=$PATH:/usr/local/go/bin
go version
```

### Need to update vendored dependencies

```bash
# Update a specific package
go get github.com/some/package@latest

# Re-vendor everything
go mod tidy
go mod vendor

# Test build
go build -mod=vendor -o /dev/null ./cmd/sling/

# Commit
git add -A
git commit -m "Update vendored dependencies"
git push origin main
```
